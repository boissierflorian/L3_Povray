les motifs à liste

Ces motifs ne fonctionnent pas avec une palette de gradation, mais avec une (petite) liste de valeurs successives.

Remarque : le passage abrupt d'une valeur à l'autre dans ces motifs les rend inutiles pour les normales et les isosurfaces.

brick

POV-Ray 3.0

brick combine deux éléments, un pour la surface des joints, et un pour la surface de la brique. Le mot clé mortar suivi d'un nombre décimal spécifie l'épaisseur du joint de mortier, et le mot clé brick_size suivi d'un vecteur spécifie la dimension de chaque brique. Les valeurs par défaut sont <8,3,4.5> et 0.5. 

#declare Mur = box {... pigment {brick color DarkRed color Gray50 mortar .2}}

Attention, toutefois, ce motif fonctionne bien sur un simple pan de mur, mais donne des résultats peu satisfaisants si le mur change de direction (un angle droit par exemple).

Utilisé dans un pigment, ce motif possède, si on en spécifie pas, une liste de couleurs particulières.

checker

POV-Ray 2.0

'Ze grand classique du raytracing ! checker fait alterner des cubes (de 1 unité de côté) de deux sortes, à la manière d'un échiquier. 

#declare Sol = plane {y,0 pigment {checker color Black color White scale 5}}

Utilisé dans un pigment, ce motif possède, si on en spécifie pas, une liste de couleurs particulières.

hexagon

POV-Ray 2.0

Ce motif est une répétition de trois zones hexagonales visibles dans le plan XZ. Perpendiculairement à ce plan, les hexagones s'étendent indéfiniment. L'hexagone situé à l'origine prend la valeur du premier élément de la liste.

#declare Pavage = plane {y,0 pigment {hexagon color Yellow color Blue color Black scale 15}}

Utilisé dans un pigment, ce motif possède, si on en spécifie pas, une liste de couleurs particulières.

object

POV-Ray 3.5

Ce motif, pour lequel on doit spécifier un objet solide, prend la valeur 0 pour les points à l'extérieur de cet objet, et la valeur 1 pour les points à l'intérieur de cet objet.

Utilisé dans un pigment, ce motif possède, si on en spécifie pas, une liste de couleurs particulières.

remarque

Si un de ces motifs est utilisé dans une définition de media, il est obligatoire de spécifier des density, comme ceci :

media {
  emission .05
  density {
   checker   
    density {rgb 0}
    density {rgb Red}
    }
  }
