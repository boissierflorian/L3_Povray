prise en main de POV-Ray pour Linux - installation améliorée

Pour pouvoir utiliser POV-Ray de manière plus efficace, un certain nombre de petites adaptations à votre environnement peuvent être utiles.

Accéder à POV-Ray depuis n'importe quel répertoire

Pour cela il faut placer l'exécutable dans un répertoire accessible, ou rendre accessible le répertoire dans lequel se trouve l'exécutable. Trois solutions s'offrent à vous.

Copier l'exécutable dans un endroit accessible

Vous pouvez placer l'executable dans un répertoire tel que /bin ou /usr/bin. Cela peut être source de désordre dans votre arborescence (on est pas sous Windows tout de même) et pas vraiment conseillé. Il vaut mieux le mettre dans /usr/local/bin/ (et verifier que ce repertoire est accessible). 

root$ cp /usr/local/lib/povray-3.5/povray /usr/local/bin/

Mettre les pages de manuel a leur place (accessibles par la commande man povray) :

root$ mv povray.1 /usr/local/man/man1

faites vous maintenant un répertoire (Sous votre compte utilisateur):

root$ exit
zorro$ cd ~/
zorro$ mkdir povray
zorro$ cd povray
zorro$ cp /usr/local/lib/povray/scenes/advanced/skyvase.pov ./


zorro$ povray -Iskyvase.pov -L/usr/local/lib/povray31/include/ +V +D -w320 -h240 +FT +Ovase.tga

...devrait lancer le rendu de la scene.

Adapter la variable d'environement PATH

La deuxième solution est d'ajouter le répertoire de POV-Ray a votre PATH (la variable d'environement qui contient tous les repertoires accessibles), grâce à la commande «export» pour bash ou la commande «setenv» pour tcsh: 

zorro$ export PATH="$PATH:/usr/local/lib/povray31/"
zorro$ setenv PATH "$PATH:/usr/local/lib/povray31/"

Créer un lien symbolique

Troisieme solution (et la plus élégante), le lien symbolique. C'est en gros un «racourci» vers le vrai executable

Vérifiez que le repertoire /usr/local/bin est dans le PATH

zorro$ echo $PATH

...vous devriez obtenir l'information suivante :

/bin:/usr/bin:/usr/X11R6/bin:/usr/local/bin:...

sinon, ajoutez /usr/local/lib/ dans le PATH en faisant :

zorro$ export PATH="$PATH:/usr/local/bin"
zorro$ setenv PATH "$PATH:/usr/local/bin"

Pour que cette derniere commande soit permantente, il faut la rajouter dans votre .bashrc ou votre .tcshrc

Puis repassez en root (su) et faites :

root$ cd /usr/local/bin/
root$ ln -s /usr/local/lib/povray-3.5/povray ./povray

Désormais vous n'avez plus besoin d'etre root. il est donc (très vivement) conseillé de revenir en mode utilisateur par la commande exit 

root$ exit

chemin des fichiers include

Si on essaye de lancer le rendu avec :

zorro$ povray -Iskyvase.pov +V +D -w320 -h240 +FT +Ovase.tga

On n'obtient qu'un message d'erreur en guise de remerciement :

...
Parsing...Could not find file 'shapes.inc'
...

Ce qui signifie en bon français que POV-Ray n'arrive pas a trouver le fichier shapes.inc qui lui est pourtant necessaire. Il faut alors lui indiquer son emplacement comme précédemment par une option : 

-L/usr/local/lib/povray-3.5/include

Pour ne pas surcharger les lignes de commande et avoir à retaper 36000 fois la même chose, on va definir une bonne fois pour toutes ces parametres dans un fichier de configuration. Ce fichier se nomme .povrayrc (le . indique que c'est un fichier caché, visible par ls -a) et se place dans le home de l'utilisateur.

Une ébauche de fichier de configuration est fournie avec POV-Ray, il suffit de la copier :

zorro$ cp /usr/local/lib/povray-3.5/ini/povray.ini ~/.povrayrc

Il ne reste qu'à éditer ce fichier et à le configurer selon votre convenance. Les commentaires permettent de comprendre aisement le fonctionnement de tous les paramètres. Ils ne sont cependant pas tous listés.

Voila de quoi commencer avec pov pour linux. Pour essayer cette nouvelle mouture, allez faire un tour dans les scenes fournies en exemple /usr/local/lib/povray-3.5/scenes/.
Amusez-vous bien.
