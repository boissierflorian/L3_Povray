sphere_sweep ("glissé de sphère" !)

objectif

Créer des objets filiformes avec une épaisseur variable.

compatibilité

POV-Ray 3.5 +sup.

explication

Une sphere_sweep est créée à partir du volume occupé par le parcours d'une sphère le long d'une spline. Le rayon de la sphère peut varier aux points de contrôle de la spline. Une application typique : les câbles !

syntaxe

sphere_sweep {
  linear_spline / b_spline / catmull_rom_spline
  Nombre_Spheres,       // entier
  Centre_1, Rayon_1,    // vecteur, décimal 
  Centre_2, Rayon_2,    
  ...                   
  Centre_n, Rayon_n     
  (tolerance Tolerance) // décimal

  (..modificateurs d'objets..)
  (..transformations..)
  (..matériau..)
}

Il n'est pas possible de déclarer une spline pour l'utiliser dans votre objet sphere_sweep, car une spline traditionnelle ne contient pas l'information du rayon aux points de contrôle.

Il faud d'abord spécifier le type de spline à employer (les images suivantes sont construites avec des points de contrôle identiques) :

- linear_spline


- b_spline


- catmull_rom_spline


La linear_spline, porte bien son nom. Elle se contente en effet de relier les sphères avec des sortes de cônes. On constate que la b_spline ne passe pas exactement sur les points, tandis que la catmull_rom_spline le fait.

Si le rendu de la sphere_sweep montre quelques imperfections, il est nécessaire d'écrire le mot-clef tolerance suivi de la nouvelle tolérance (la valeur par défaut est 0.000001). Essayez une valeur telle que 1.0e-4 ou plus, ça devrait jouer...

exemple

// La spline ci-dessus en version b_spline
sphere_sweep {
  b_spline,
  11,
  < -10,1, 0>, 0.1
  < -8, 3, 0>, 0.2
  < -6, 4, 0>, 0.1
  < -4, 2, 0>, 0.2
  < -2, -3, 0>, 0.1
  < 0,  0, 0>, 0.2
  < 2,  -4, 0>, 0.1
  < 4,  3, 0>, 0.2
  < 6,  4, 0>, 0.1
  < 8,  -1, 0>, 0.2
  < 10, 0, 0>, 0.1
  tolerance 1.0e-4
}

CSG

Fonctionne bien en CSG, l'intérieur et l'extérieur étant bien définis. Il est cependant déconseillé d'utiliser une sphere_sweep avec une opération de mélange (merge), de petits trous pourraient se présenter entre les objets mélangés.

remarques

- Si les taches noires persistent (on dirait une pub pour de la lessive), essayez de changer la méthode d'anticrénelage, en ajoutant Sampling_Method=2 (+AM2) dans le fichier .ini ou la ligne de commande.

- Souvent, il est plus intéressant d'utiliser une fonction spline pour répartir plein de sphères et de cônes que d'utiliser sphere_sweep. En pratique, seul un problème de consommation de mémoire vous fera choisir cette dernière.
