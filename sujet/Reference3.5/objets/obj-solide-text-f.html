l'objet texte

objectif

créer du texte en 3D

compatibilité

POV-Ray 2.0 +sup.

syntaxe

text {
        ttf "nomdelapolice.ttf"      // éventuellement chemin complet
        "Chaîne de caractères"       // caractètes alphanumérique
        Epaisseur ,                  // nombre décimal
        Espacement                   // vecteur
        (..modificateurs d'objets..)
        (..transformations..)
        (..matériau..)
       }

Le texte sans épaisseur, autrement dit la face des caractères commence à l'origine et se developpe le long de l'axe +x. L'épaisseur se fait le long de l'axe +z

Seules les polices TrueType fonts (ttf) et TrueType Collections (ttc) sont supportés.

L'espacement horizontal (crénage) est géré en interne par POV-Ray en incluant les informations de la police. Le vecteur obligatoire "Espacement" peut définir un déplacement supplémentaire entre chaque caractère. En général "Espacement" est à de <0,0,0>, mais on ne peut décaler les caractères sur les axes X et Y (Z sera ignoré). Par exemple, spécifier 0.1*x ajoutera 0.1 unité d'espace entre chaque caractère.

En principe, une unité d'espace vertical correspondra à la hauteur d'une ligne de texte, interligne comprise (à condition que la fonte soit réalisée rigoureusement). Un caractète utilise 50 à 75% d'un unité de hauteur.

Seuls les caractères imprimables sont utilisables, donc pas de retour chariot, saut de ligne et autres tabulations.

Pour accéder facilement à vos fontes, spécifiez le chemin dans le fichier "povray.ini". par exempe: Library_Path=C:\WINDOWS\Fonts

exemple

 
text { ttf "arial.ttf" "text" 0.4, <0,0,0> pigment {Yellow}}

CSG

Fonctionne parfaitement en CSG, l'intérieur et l'extérieur étant bien définis.

remarque

Afin d'obtenir un rendu correct des caractères accentués (pour ne pas avoir l'accent tué), il faut, dans les global_settings, paramétrer charset avec utf8.

global_settings { charset utf8 }

macros standard (POV-Ray 3.5)

Dans "shapes.inc" :

Bevelled_Text(Font, String, Cuts, BevelAng, BevelDepth, Depth, Offset, UseMerge)
Crée du texte 3D chanfreiné.

Font         =   nom de la police
String       =   chaîne de caractères
Cuts         =   Le nombre de coupe qui permet d'obtenir le chanfrein (nb élevé = rendu plus long)
BevelAng     =   Angle du chanfrein en degré
BevelDepth   =   Profondeur du chanfrein 
Depth        =   Epaisseur
Offset       =   Espacement
UseMerge     =   0: merge ; 1: pas merge

Circle_Text(Font, Text, Size, Thickness, Radius, Inverted, Justification, Angle)
Crée du texte avec le haut ou la base des caractères aligné sur un arc de cercle.
Font          =  nom de la police
Text          =  chaîne de caractères
Size          =  Taille des caractères, agit comme scale
Thickness     =  Epaisseur
Radius        =  Rayon du chemin circulaire
Inverted      =  0: bases alignés ; 1: hauts alignés
Justification =  1: Alignement gauche ; 2: Alignement Droite ; 3: Alignement Centré
Angle         =  Angle à partir duquel le chemin circulaire commence (0° sur l'axe x et 90° sur l'axe y)

TextWidth( Font, Text, Size )
Retourne la largeur d'un texte spécifique dont on connait la police et la valeur de l'échelle employée. Cette macro peut être utilisée pour une mise en position de textes et d'objets. Comme cette macro tient compte des espaces pouvant être placés avant et/ou après la chaîne de caractères, il se peut que la largeur retournée soit plus grande que la largeur retournée par les fonctions min_extent and max_extent de la "bounding box" de cette même chaîne de caractères.
Font     =  nom de la police utilisée.
Text     =  chaîne de caractères pour laquelle on désire connaître la largeur.
Size     =  dimension de l'échelle à laquelle sont mis les caractères.
