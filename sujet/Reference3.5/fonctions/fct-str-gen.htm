chaînes de caractères

Une chaîne de caractère est... une suite de caractères ! Des lettres, des chiffres, des signes,... bref, (presque) tout se qui se trouve dans une table de caractères ASCII. Elles peuvent servir à spécifier un nom de fichier, à donner un texte à rendre avec l'objet text, à passer un message sur la console, etc...

syntaxe

Une chaîne de caractères est toujours placée entre des guillemets, et peut contenir jusqu'à 256 caractères.

"je suis une chaîne de caractères"

On peut la déclarer dans une variable.

#declare Chaine1 = "le rapide renard brun saute par dessus le chien paresseux"

Parfois, pour éviter une confusion de syntaxe, il faut recourir à des caractères spéciaux pour signifier certains caractères.


\" place un guillemet dans la chaîne

... Chaine2 = "un \"batiscaphe\" entre guillemets"  

donnera : un "batiscaphe" entre guillemets



\\ place un backslash dans la chaîne (sert à éviter la confusion avec le dernier guillemet)

... Chaine2 = "..\\maison\\cuisine\\tiroir\\fourchette"  

donnera : ..\maison\cuisine\tiroir\fourchette

L'option charset dans les global_settings permet de spécifier le jeu de caractères employé dans les chaînes de caractères. Vous avez actuellement le choix entre ascii et utf8 (unicode).

Si vous avez choisi utf8, un caractère Unicode se place dans une chaîne comme ceci : \uNNNN, NNNN étant le numero à quatre chiffres (en hexadécimal) que vous pouvez trouver dans les tables unicode. Exemple pour "sphère" :

#declare MaChaine = "sph\u00E8re"

Notez cependant que les noms de fichiers restent strictement interprétés en ASCII. Il est donc impossible de traiter dans POV-Ray des fichiers dont le nom comporte des accents. Évitez également, autant que possible, de spécifier des chemins d'accès dans la châine d'un nom de fichier, utilisez l'option Libray_Path (+L) pour cela.
