hf_gray_16

objectif

Faire en sorte que l'image de sortie (rendu) soit utilisable par POV-Ray lui-même, dans une autre scène, sous forme d'un height-field dont les hauteurs sont codées sur 16 bits (65 535 niveaux).

(souvent, on pointera la caméra sur un simple plan texturé, afin de profiter de la richesse du système de textures de POV-Ray)

compatibilité

POV-Ray 3.0 +sup.

syntaxe

global_settings {
        hf_gray_16 (on/off)       
        }

Le booléen (on/off) est optionnel, l'option est activée par hf_gray_16 seul.

Si l'option hf_gray_16 est activée, le fichier de sortie pourra être utilisé directement dans un height_field. L'image apparaitra en noir et blanc dans la fenêtre de visualisation, plutôt qu'en couleurs. Ceci permet d'appréhender plus facilement l'aspect général du height_field, car le codage particulier de l'image rend sa visualisation difficile (sauf si la format de sortie est le PNG).

exemple

 
//un HF "vallonné"...
global_settings { hf_gray_16 on }
camera {
	orthographic
	location <0, 20, 0>
	right 5*x
	up 5*y
	look_at <0, 0, 0>
}
plane {
	y, 0
	texture {
		pigment {
			bozo
			color_map {
				[0 rgb 0]
				[1 rgb 1]
			}
		}
		finish { ambient 1 }
	}
