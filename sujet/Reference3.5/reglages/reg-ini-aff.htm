options d'affichage

modes graphiques

commandeinidéfautdescription
+D, -D Display=b - enclenche l'affichage progressif de l'image calculée
+Dn, -Dn Video_Mode=n 0 sélectionne le mode d'affichage à utiliser
+Dnx, -Dnx Palette=x ?? sélectionne une palette d'affichage
(inexistant) Display_Gamma=n.n ?? détermine le gamma d'affichage

Le choix du mode d'affichage n'a de sens que pour les versions "console" de POV-Ray, celles fonctionnant en environnement graphique utilisant le mode de l'environnement. Une valeur de 0 (+D0) provoque la sélection automatique du mode d'affichage.

Pour ce qui concerne le gamma, voyez la section éponyme dans les "concepts généraux".

divers

commandeinidéfautdescription
+P, -P Pause_When_Done=b - une fois l'image terminée, se mettre en pause
+V, -V Verbose=b - enclenche l'affichage des messages (warnings, statistics,..)
+UD, -UD Draw_Vistas=b - enclenche le tracé des vista_buffers

Le "vista buffer" est un système d'optimisation des calculs, qui consiste à projeter dans le plan de l'image les limites des "boîtes d'optimisation", sous forme de rectangles hiérarchisés. Cela permet à POV-Ray d'éviter de lancer des tests d'intersection en 3D si le "buffer" 2D d'un objet donné ne comprend même pas le pixel qui est calculé. L'option +UD permet de visualiser ces rectangles, en bleu et en rouge. Attention, le "vista buffer" ne marche pas dans les modes de perspectives "déformants" (où les droites ne restent pas droites).

commandeinidéfautdescription
+SPn Preview_Start_Size=n 1taille de départ des éléments de la mosaïque
+EPn Preview_End_Size=n 1taille finale des éléments de la mosaïque

Ces options permettent, tout en gardant la résolution de l'image, d'effectuer des rendus grossiers mais rapides, avec un raffinement successif. Le rendu sera constitué de gros pixels, dont la taille initiale (+SPn) doit être une puissance de 2 : 1, 2, 4, 8, 16, 32,... Ensuite, le rendu reprend automatiquement en utilisant une taille de pixel deux fois plus petite que la précédente, jusqu'à atteindre la taille de pixel spécifiée avec +EPn. Si cette dernière taille est supérieure à 1, POV-Ray finira avec un rendu d'une finesse de 1 pixel. Attention, seule la dernière image sera sauvegardée.
