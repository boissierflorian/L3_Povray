Glow

objectif

Créer des points luminescents.

compatibilité

MegaPOV 1.0

syntaxe

glow {
  type (0 / 1 )
  location Position    // vecteur
  size Taille          // décimal
  radius Rayon         // décimal
  (fade_power Exposant)// décimal
  color Couleur        // vecteur de couleur

  (..transformations..)
}

Un glow, c'est un peu comme une sphère transparente avec un point incandescent en son centre. Le rayon de la 'sphère' est spécifié avec radius, son centre avec location. L'intensité du point central est donnée par le mot clé size. Selon le type et le fade_power (fondu), on aura un dégradé d'une certaine allure entre le centre et le pourtour apparent de la sphère.

Grâce à la documentation exhaustive de Chris Huff, il ne m'est pas possible de vous dire quelle est la différence entre le type 0 et le type 1. Visuellement, je n'ai pas vu de différences. Ne mettez rien, c'est plus simple.

fade_power permet de spécifier l'allure du dégradé, sa vitesse en quelque sorte. Une valeur 1 (par défaut) donnera un dégradé linéaire, une valeur 2 donnera un dégradé plus "serré".

Remarque : selon le fondu et l'intensité, certains glows ne sont pas totalement transparents au bord de la sphère, rendant son contour apparent. Un fade_power de 2 ou plus permet généralement d'éviter ce problème.

Enfin, on peut spécifier une couleur pour le glow, qui est blanc par défaut.

On peut "attacher" un glow à une source lumineuse, comme ceci :

light_source {
<50,40,30> White
glow {size 1 radius 20}
}

Pas besoin de spécifier une position et une couleur, le glow prendra automatiquement celles de la source lumineuse, même si elle est transformée.

exemple

#declare I=0;
#while (I < 1)
 glow {
  location <50,0,0>
  color SteelBlue*0.5
  size 1.5
  radius 20
  fade_power 2
  rotate y*360*I
 }
#declare I=I+.1;
#end

remarques

Les glows ne peuvent pas faire partie d'une union, ni d'aucune opération de CSG. Sauf si c'est un glow attaché à une source lumineuse, et que cette source fait partie d'une union (pour faire une LED, par exemple).
